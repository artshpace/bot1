# SPEC 1.1.1 — МАГАЗИН И ПОКУПКИ

## 1. МАГАЗИН В ЛИЧНОМ КАБИНЕТЕ

### Новый раздел "Магазин"

Раздел должен быть доступен для:
- [ ] Ученика
- [ ] Родителя
- [ ] Преподавателя

### Что показывать в магазине

- [ ] Абонементы (по направлениям)
- [ ] Онлайн-курсы
- [ ] Мастер-классы
- [ ] Интенсивы
- [ ] Сертификаты
- [ ] Мерч

### Требования

- [ ] Каждый товар имеет цену
- [ ] Каждый товар имеет описание
- [ ] Каждый товар имеет кнопку "Купить" или "В корзину"
- [ ] Товары отсортированы по категориям
- [ ] Видно наличие товара
- [ ] Видна примерная дата доставки (если applicable)

---

## 2. КАТАЛОГ АБОНЕМЕНТОВ

### Текущая проблема

- Могут отображаться несуществующие абонементы
- Дублирование товаров
- Тестовые продукты видны пользователю

### Требования

- [ ] Провести полный аудит всех абонементов
- [ ] Удалить несуществующие абонементы
- [ ] Удалить дубликаты
- [ ] Убрать тестовые товары из публичного каталога

### Пример: Гитара

Если по гитаре существует **только** абонемент на 12 занятий:
- [ ] Показывать только этот абонемент
- [ ] Запретить отображение других вариантов
- [ ] Запретить отображение тестовых опций

### Единый каталог

- [ ] Все абонементы в одном месте в БД
- [ ] Все абонементы имеют статус (active, inactive, draft)
- [ ] Активны только товары со статусом `active`
- [ ] Невозможно заказать товар со статусом `inactive`

### Структура абонемента

```
{
  id: string (UUID)
  direction: "guitar" | "acting" | "vocals" | "dance" | "painting"
  title: string
  description: string
  price: number (в тенге)
  duration: number (количество занятий или недель)
  duration_type: "sessions" | "weeks" | "months"
  status: "active" | "inactive" | "draft"
  created_at: timestamp
  updated_at: timestamp
}
```

## 3. ПОКУПКА АБОНЕМЕНТА

### Текущая проблема

- Кнопка "Купить абонемент" не запускает полноценный процесс покупки

### Новый процесс покупки

1. Пользователь нажимает "Купить" или "В корзину"
2. Товар добавляется в корзину
3. Пользователь переходит в корзину или продолжает покупки
4. Нажимает "Оформить заказ"
5. Выбирает способ оплаты:
   - Карта (Kaspi, KKBP, другие)
   - Электронный кошелёк (Paybox, другие)
6. Создаётся заказ
7. Пользователь перенаправляется к платёжному шлюзу
8. После успешной оплаты:
   - Заказ отмечается как оплачен
   - Абонемент активируется
   - Отправляется письмо с подтверждением
   - Отправляется уведомление в Telegram (если привязан)

### Требования

- [ ] Без оплаты ничего не активировать
- [ ] Заказ создаётся только после подтверждения платежа
- [ ] Абонемент активируется автоматически после оплаты
- [ ] Сохраняется информация об оплате (дата, сумма, способ)

### Ошибки и отмены

- [ ] Если платёж не прошёл, заказ остаётся со статусом `pending`
- [ ] Пользователь может повторить попытку
- [ ] Пользователь может отменить заказ до оплаты (удалить из корзины)
- [ ] Если оплата отменена пользователем, заказ не создаётся

## 4. КОРЗИНА

### Текущая проблема

- Корзина может быть общей между пользователями
- Товары из других аккаунтов могут быть видны

### Требования

- [ ] Корзина принадлежит конкретному пользователю (хранится в БД с user_id)
- [ ] После регистрации нового пользователя его корзина пуста
- [ ] Товары одного пользователя не видны другому
- [ ] После выхода гостевая корзина очищается
- [ ] После входа загружается персональная корзина пользователя
- [ ] При смене пользователя корзина меняется
- [ ] Можно добавлять товары в корзину без авторизации (гостевая корзина)
- [ ] При авторизации гостевая корзина удаляется

### Структура корзины

```
{
  id: string (UUID)
  user_id: string | null (null для гостей)
  items: [
    {
      id: string (UUID)
      product_id: string
      quantity: number
      price: number (цена в момент добавления)
    }
  ]
  created_at: timestamp
  updated_at: timestamp
}
```

### Функциональность

- [ ] Добавить товар в корзину
- [ ] Удалить товар из корзины
- [ ] Изменить количество товара
- [ ] Очистить корзину
- [ ] Просмотр корзины с итоговой суммой
- [ ] Сохранение корзины при обновлении страницы

## 5. ЗАКАЗЫ

### Требования

- [ ] Существуют реальные заказы в БД
- [ ] Заказы связаны с конкретным пользователем (user_id)
- [ ] Заказы сохраняются после перезагрузки
- [ ] История заказов доступна в ЛК пользователя

### Статусы заказа

- `pending` — В ожидании оплаты
- `paid` — Оплачен
- `processing` — Обработка
- `completed` — Завершён
- `cancelled` — Отменён
- `refunded` — Возврат

### Структура заказа

```
{
  id: string (UUID)
  user_id: string
  status: "pending" | "paid" | "processing" | "completed" | "cancelled" | "refunded"
  items: [
    {
      product_id: string
      quantity: number
      price: number
    }
  ]
  total: number
  payment_method: string
  payment_id: string | null
  created_at: timestamp
  paid_at: timestamp | null
  completed_at: timestamp | null
}
```

### История заказов

- [ ] В ЛК отображается список всех заказов пользователя
- [ ] Для каждого заказа видны товары, сумма, дата, статус
- [ ] Пользователь может посмотреть детали заказа
- [ ] Пользователь может скачать квитанцию (PDF)

## 6. АКТИВНЫЕ АБОНЕМЕНТЫ

### В ЛК пользователя (вкладка "Абонементы" или "Мои курсы")

- [ ] Список активных абонементов
- [ ] Дата истечения абонемента
- [ ] Количество оставшихся занятий
- [ ] Кнопка "Продлить" (если абонемент заканчивается)
- [ ] Кнопка "Отменить" (если применимо)

### Активация абонемента

После оплаты заказа:
- [ ] Абонемент появляется в ЛК автоматически
- [ ] Дата активации = дата оплаты
- [ ] Дата истечения = дата активации + срок абонемента

## 7. ВОЗВРАТЫ И ОТМЕНЫ

### Требования

- [ ] Пользователь может отменить заказ (если статус не `completed`)
- [ ] Отмена возвращает товар в каталог
- [ ] Проверить, должна ли быть комиссия при отмене
- [ ] При возврате деньги возвращаются пользователю (на счёт, карту)
- [ ] Операция возврата логируется

### Информация о платёжной системе

**Требуется уточнение:**
- [ ] Какой платёжный шлюз используется?
- [ ] Какой API используется?
- [ ] Есть ли API-ключи?
- [ ] Есть ли test mode и live mode?
- [ ] Как обрабатываются webhooks от платёжной системы?

### Требования к интеграции

- [ ] После успешного платежа есть callback от платёжной системы
- [ ] Статус заказа обновляется автоматически
- [ ] Абонемент активируется автоматически после callback
- [ ] Если webhook не пришёл, есть механизм проверки статуса платежа
- [ ] Невозможны дублирующиеся активации одного заказа

## 9. ПИСЬМА О ПОКУПКАХ

После успешной оплаты:

- [ ] Отправляется письмо с подтверждением заказа
- [ ] Письмо содержит номер заказа
- [ ] Письмо содержит список товаров
- [ ] Письмо содержит сумму
- [ ] Письмо содержит дату и время заказа
- [ ] Письмо содержит ссылку на страницу заказа в ЛК

После успешной оплаты (если пользователь привязал Telegram):

- [ ] Отправляется уведомление в Telegram

- [ ] Уведомление содержит информацию об активированном абонементе
