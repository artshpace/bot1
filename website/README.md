## Зависимости

**Никаких** — браузер и любой локальный HTTP-сервер.

> Важно: открывать файлы через `file://` нельзя — шрифты Google Fonts и относительные пути к изображениям требуют HTTP-сервера.

## Запуск локально

Запускать сервер нужно из **корня репозитория** (`bot1/`), а не из папки `website/`. Иначе пути к изображениям (`../media/`) не разрешатся.

### Вариант 1 — Python (рекомендуется, есть на macOS/Linux)

```bash
cd /путь/к/bot1
python3 -m http.server 8000
```

Открыть в браузере: [http://localhost:8000/website/](http://localhost:8000/website/)

### Вариант 2 — Node.js (npx serve)

```bash
cd /путь/к/bot1
npx serve . -p 8000
```

Открыть: [http://localhost:8000/website/](http://localhost:8000/website/)

### Вариант 3 — VS Code Live Server

1. Установить расширение **Live Server** (Ritwick Dey).
2. Открыть папку `bot1/` в VS Code.
3. Правый клик на `website/index.html` → **Open with Live Server**.
4. Сервер запустится от корня репозитория автоматически.

### Вариант 4 — Node.js (http-server)

```bash
npm install -g http-server
cd /путь/к/bot1
http-server . -p 8000
```

## Личный кабинет (v0.2)

Система авторизации и личный кабинет ученика. Реализованы:

- **Авторизация:** регистрация по телефону или email, вход, восстановление доступа, защищённые маршруты (редирект на вход для неавторизованных).
- **Кабинет ученика:** имя, направление, преподаватель, дата ближайшего занятия, расписание на неделю, остаток занятий, срок абонемента, статус оплаты.
- **Мои курсы:** список приобретённых онлайн-курсов с прогрессом в процентах и кнопкой перехода.
- **Расписание:** календарное отображение занятий по месяцам с детализацией дня.

### Демо-доступ

Без регистрации можно войти под демо-аккаунтом:

```
Логин:  demo@shpigotskiy.art
Пароль: demo1234
```

### Как это устроено (важно для интеграции)

> Реального бэкенда нет — используются **моковые данные и mock-API** на стороне браузера.

- `js/api.js` — единственная точка доступа к данным. Все страницы вызывают `window.API.*`, который сейчас возвращает Promise с тестовыми данными и хранит пользователей/сессию в `localStorage` (`sas_users`, `sas_session`). Когда появится настоящий backend — меняется **только этот файл** (методы заменяются на `fetch(...)`), код страниц трогать не нужно.
- `js/auth.js` — защита маршрутов и управление сессией. Подключается до `account.js`, выполняется немедленно: неавторизованных перенаправляет на `login.html`.
- `js/account.js` — логика всех страниц кабинета (один файл, секции выполняются только при наличии нужных элементов).
- `css/account.css` — стили кабинета, переиспользуют переменные из `style.css`.

## Абонементы, платежи и админ-панель (v0.3)

Управление абонементами, платежами и базовое администрирование. Реализованы:

- **Абонементы** (сущность `Абонемент`: название, количество занятий, стоимость, срок действия, статус `активный/завершён/заморожен`). В кабинете на главной показываются текущий абонемент, остаток занятий, дата окончания и статус.
- **Мои абонементы** — история всех абонементов ученика (название, дата покупки, дата окончания, количество занятий, статус).
- **Платежи** (сущность `Платёж`: дата, сумма, назначение, статус). Раздел «История платежей» в кабинете.
- **Покупка** — интерфейсы покупки абонемента, продления абонемента и покупки онлайн-курса (страница «Оплата и покупки»). Реальная оплата не подключена; все покупки проходят через единую функцию-заглушку `processCharge()` — точку будущей интеграции платёжных систем.
- **Уведомления** в кабинете (без email/SMS): осталось менее 3 занятий, абонемент заканчивается менее чем через 7 дней, есть неоплаченные услуги.
- **Админ-панель** (роль `admin`, защищённый раздел): ученики (список, поиск, карточка), абонементы (создание/редактирование/удаление), курсы (создание/редактирование/удаление), платежи (просмотр).

### Демо-доступ

```
Ученик    — demo@shpigotskiy.art  / demo1234
Админ     — admin@shpigotskiy.art / admin1234
```

Админ после входа попадает в админ-панель (`admin.html`); ученики — в свой кабинет.

### Что добавилось в архитектуре

- **Новые namespace в `API`:** `subscriptions`, `payments`, `notifications`, `admin`; расширены `courses` (каталог + покупка) и `student.profile` (данные берутся из активного абонемента).
- **Данные** хранятся в `localStorage`: `sas_subscriptions`, `sas_payments`, `sas_courses` (плюс прежние `sas_users`, `sas_session`). Сид-данные создаются один раз; апгрейд с v0.2 происходит автоматически.
- **Роли:** `auth.js` различает публичные, пользовательские и админ-страницы; ученик не может открыть админ-раздел.
- **Боковое меню** кабинета и админки рендерится из `account.js` (`[data-cab-sidebar]`) — навигация в одном месте, без дублирования по страницам.
- **Точка интеграции оплаты** — `processCharge(order)` в `api.js`: сейчас возвращает успешный мок, позже заменяется вызовом реального провайдера (Kaspi / CloudPayments / Stripe …).

### Архитектура на будущее (НЕ реализовано)

Заготовлены, но намеренно не реализованы — добавляются в следующих версиях:

- **Реальные платежи** — через `processCharge()` (см. выше).
- **Родительский кабинет** — зарезервирован `API.parent`, в меню помечен «v0.4».
- **Посещаемость** — зарезервирован `API.attendance`.
- **Домашние задания** — зарезервирован `API.homework`, в меню помечен «v0.4».
- **Сертификаты** — зарезервирован `API.certificates`, в меню помечен «v0.4».
- **Telegram Mini App** — модель пользователя (`role`, единый mock-API) совместима с будущей авторизацией через Telegram; сам Mini App живёт отдельно в `bot1/index.html`.

Эти разделы видны в боковом меню кабинета с пометкой «Скоро», их методы в `api.js` сейчас возвращают понятную ошибку-заглушку.
