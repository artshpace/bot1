# SPEC 1.1.1 — АВТОРИЗАЦИЯ И АККАУНТЫ

## 1. РЕГИСТРАЦИЯ И ПОДТВЕРЖДЕНИЕ EMAIL

### Текущая проблема

- После регистрации пользователь автоматически входит в аккаунт
- Подтверждение почты отсутствует

### Новый сценарий

1. Пользователь создаёт аккаунт
2. Система создаёт пользователя
3. Отправляет письмо подтверждения
4. Пользователь подтверждает email по ссылке в письме
5. Только после подтверждения может войти

### Статусы email

- `unconfirmed` — Email не подтверждён
- `confirmed` — Email подтверждён

### Требования

- [ ] Запретить вход без подтверждения email
- [ ] Письмо содержит уникальную ссылку подтверждения
- [ ] Ссылка имеет срок действия (по умолчанию 24 часа)
- [ ] После подтверждения статус меняется на `confirmed`
- [ ] Может ли пользователь запросить повторное письмо подтверждения?

- `student` — Ученик
- `parent` — Родитель
- `teacher` — Преподаватель

- [ ] По умолчанию новый пользователь получает роль по его выбору
- [ ] Роль сохраняется в БД

## 6. ВОССТАНОВЛЕНИЕ ПАРОЛЯ

### Текущая проблема

- Письмо восстановления может не отправляться

### Требования

- [ ] Работает ли восстановление пароля?
- [ ] Создаётся ли токен восстановления?
- [ ] Отправляется ли письмо восстановления?
- [ ] Письмо содержит ссылку с токеном?
- [ ] Ссылка имеет срок действия (по умолчанию 1 час)
- [ ] После перехода по ссылке отображается форма установки нового пароля
- [ ] Новый пароль требует подтверждения
- [ ] После установки токен становится невалидным
- [ ] Добавить функцию "Показать / Скрыть пароль" при восстановлении

### Процесс

1. Пользователь вводит email на странице восстановления
2. Система проверяет существование пользователя
3. Система создаёт токен восстановления (уникальный, с TTL)
4. Система отправляет письмо с ссылкой `https://domain/restore-password?token=...`
5. Пользователь переходит по ссылке
6. Система проверяет валидность токена
7. Пользователь вводит новый пароль (дважды)
8. Система обновляет пароль и инвалидирует токен
9. Пользователь может войти с новым паролем

## 7. СОХРАНЕНИЕ СЕССИИ

### Текущая проблема

- Пользователь может выходить после перехода на главную страницу
- Авторизация может не сохраняться после обновления страницы

### Требования

- [ ] Авторизация сохраняется после обновления страницы
- [ ] Авторизация сохраняется после переходов между страницами
- [ ] Пользователь не выходит автоматически при переходе
- [ ] Сессия имеет разумный TTL (по умолчанию 7 дней)
- [ ] При истечении TTL пользователь должен переавторизоваться

### Реализация

- Использовать JWT или session ID в http-only cookie
- Не хранить токены в localStorage
- При загрузке страницы проверять валидность сессии
- При необходимости рефрешить токен

## 9. СИСТЕМА РОЛЕЙ И ПРАВ

### Где проверяются права

- [ ] На сервере (обязательно)
- [ ] На клиенте (для UX)

### Ограничения по ролям

Ученик:
- Не видит функции администратора
- Не видит функции преподавателя
- Не видит функции родителя

Родитель:
- Не видит функции преподавателя
- Не видит функции администратора
- Видит ЛК ученика (если связан)

Преподаватель:
- Не видит функции администратора
- Не видит функции родителя

Администратор:
- Видит все функции

### Требования

- [ ] Все права проверяются сервером
- [ ] Клиент не скрывает функцию, серверный запрос отклоняет
- [ ] Невозможно подменить роль через URL
- [ ] Невозможно подменить роль через браузер
- [ ] Невозможно получить доступ к чужому ЛК
